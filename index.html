Sobre el autor

Chris Richardson es un arquitecto de software y empresario, con grado académico en Ciencias de la Computación de la Universidad de Cambridge en Inglaterra. Es gran desarrollador en Java y autor de "POJOs in Action, que describe cómo construir aplicaciones empresariales en Java con POJOs (Plain Old Java Objects) y frameworks, como Spring e Hibernate.

Chris es el fundador de la original CloudFoundry.com, una de las primeras PaaS (Platform-As-A-Service) de Java para Amazon EC2, que fue luego adquirida por SpringSource. También es el co-fundador de Component Systems LLC., una empresa de consultoría que se enfoca en arquitectura de sistemas distribuidos de gran escala, y que fue adquirida por BEA.

Actualmente se dedica a la consultoría y a su startup eventuate.io.

Patrones base

¿Cuál arquitectura deberías elegir para una aplicación?

  • Monolítico - diseña la aplicación como una unidad desplegable individual.
  • Microservicios - diseña la aplicación como una colección de servicios ligeramente acoplados.

Descomposición

¿Cómo descomponer la aplicación en servicios?

  • Descomponer por capacidades del negocio - define los servicios de acuerdo a las capacidades del negocio.
  • Descomponer por subdominios - define los servicios de acuerdo a subdominios del modelo de dominio (Domain Driven Design)
